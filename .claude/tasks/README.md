# Tâches persistantes

Les tâches suivent le format `TASK-XXX-description.md`.

## Template

```markdown
# TASK-XXX — Titre de la tâche

**Statut** : `todo` | `in_progress` | `done` | `blocked`
**Epic** : E1 — Nom de l'epic
**User Story** : US-X.X.X

## Contexte
Pourquoi cette tâche existe.

## User Story
En tant que [rôle], je veux [action], afin de [bénéfice].

## Critères d'acceptation
- [ ] Critère 1
- [ ] Critère 2
- [ ] Critère 3

## Sous-tâches
- [ ] Sous-tâche 1
- [ ] Sous-tâche 2
- [ ] Sous-tâche 3

## Notes techniques
Détails d'implémentation, contraintes, dépendances.

## Tests
- [ ] Test unitaire : description
- [ ] Test BDD : scénario Gherkin
```
